#### Вопросы
1. Выберите один UDP-пакет. По этому пакету определите, сколько полей содержит UDP-заголовок.
   - UDP-заголовок содержит 4 поля: порт отправителя, порт получателя, длина сегмента, контрольная сумма  
   ![alt text](image.png)
2. Определите длину (в байтах) для каждого поля UDP-заголовка, обращаясь к отображаемой
   информации о содержимом полей в данном пакете.
   - Общая длина заголовка равна Length - UDP payload = 49 - 41 = 8 байт. Destination port и Source port могут иметь значения из стандартного диапазона для портов приложений 0 - 65535, т.е. по 2 байта, что соответствует значениям этих полей для данного пакета. Checksum в данном случае равно 0xbb11, это 16-битное число, отсюда можно сделать вывод, что это поле также занимает 2 байта. Тогда на поле Length остается 2 байта. Т.о., все поля имеют длину по 2 байта.
3. Значение в поле Length (Длина) – это длина чего?
   - Это длина в байтах всего UDP-сегмента, т.е. заголовка + полезных данных.
4. Какое максимальное количество байт может быть включено в полезную нагрузку UDP-пакета?
   - Т.к. поле Length - это 16-битное число, максимальное его значение равно 65535 (байт). Тогда длина полезной нагрузки может составлять до 65535 - 8 = 65527 байт.
5. Чему равно максимально возможное значение номера порта отправителя?
   - 65535 (т.к. это двухбайтовое число).
6. Какой номер протокола для протокола UDP? Дайте ответ и для шестнадцатеричной и
   десятеричной системы. Чтобы ответить на этот вопрос, вам необходимо заглянуть в поле
   Протокол в IP-дейтаграмме, содержащей UDP-сегмент.
   - 17 (0x11)  
   ![alt text](image-1.png)
7. Проверьте UDP-пакет и ответный UDP-пакет, отправляемый вашим хостом. Определите
   отношение между номерами портов в двух пакетах.
   - Порт отправителя запроса совпадает с портом получателя ответа, а порт получаетя запроса - с портом отправителя ответа.
